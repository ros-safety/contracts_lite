1.0 Contracts Lite

Provide standard practice and tooling to support the explicit definition and enforcement of function contracts. The requirements derived for this library are a subset of those specified for the C++20 contracts proposal here: http://open-std.org/JTC1/SC22/WG21/docs/papers/2018/p0542r5.html

1.1 Custom violation handling SRD001

The contracts library shall provide a mechanism to enable user-defined violation handling.

A user-defined handler allows users the ability to do custom logging or analysis of violations when they occur.

1.2 Violation handling continuation mode SRD002

The contracts library violation handling mechanism shall provide for specifying continuation mode behavior

The continuation mode determines the behavior of the handler with respect to whether it terminates the process or not. Continuation mode ON allows the process to continue exectuing, while continuation mode OFF requires the invocation of std::terminate() upon completion of the handler.

1.4 Contract enforcement level SRD004

The contracts library shall provide DEFAULT, AUDIT, and OFF enforcement levels.

The OFF enforcement level removes all checks from the compiled code. The DEFAULT enforcement level is intended to be used for contract enforcement that is intended to always run. The AUDIT level is intended for checks that are only run during extended testing and not run during normal operation.

NOTE: The term 'audit', used here in the context of contract enforcement, refers strictly to extended testing scenarios and is separate and distinct from the use of the term in the context of standards certification.
